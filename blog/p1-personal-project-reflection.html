Personal Project Reflection

Goals

  • These are the goals that I set myself for the personal project challenge.

  • - Must Have an API implemented and working correctly

  • - Elements of the page must update when button(s) are pressed

  • - Needs to be easy to use and understand + simple, sleek design

  • Stretch Goal

  • - Add in a google map that displays where the user is located

Did my Application achieve my goals?

In order for my idea to work I needed to use an API to return the information that was gathered on the users. This is the type of information that could be gathered from your machine from doing a whoIs lookup from the console. Originally I was using the geoIp API from nekudo.com but after implementation I found one major flaw. This was the fact that users could only enter in IP addresses, to someone who is not confident with computers this could be too difficult and may deter many people from using my app.
In order for me to fix this problem I needed to find other API one that would let me pass in either a domain address or an IP address. So after scouring the internet I found the solution! This API had the ability to be passed either IP or Domain addresses at the cost of only being allowed 10,000 calls an hour, this I felt would do the trick for what I'm doing.
After getting the API out of the way I then went on to creating an intro section for my users and various fields for data to be displayed on the page. These where all linked up with $("#button").on("click", function() {...}) so that when the search button is pressed some elements would disappear and new elements would take their place. This checked off the dynamic section of my goals.
Lastly I needed to Style it in a way that would make it feel modern, simple and easy to use. This was done by using a simple 4 colour scheme that I used throughout the whole project.

Completing the Stretch Goal

For my stretch goal I wanted the location for the target to be displayed on a google map with a pin marker to show the more exact location. This was something that I had tried before but couldn't seem to get working (When we worked on our personal portfolio). After doing some research I figured out how to add a map to the page and even add an API key for the map so that it will work. I then needed to update the longitude and latitude co-ordinates so that the map was showing the correct location. Lastly I added in a Pin marker and set that to the longitude and latitude co-ordinates that the map was set to.
After all this the map was working how it was supposed to but I wanted to take it a step further as I want my application to look nice. I found out that you can customize the style of a google map so looking through Snazzy maps a website that offers styles for google maps I picked out one that I liked and added it to my page!

Technical Gaps

This week I feel that I really got a better grasp on how to add API's using both an AJAX call and $getJSON call. I was able to properly get this going and had a good read through the documentation on these types of calls.
I also worked on getting elements to appear/disappear by wrapping all the elements that where to change in a div and only changing the visibility of the container div instead of each element individually. This helped clean up my JavaScript and also may it much easier to code.
I learnt how to style new elements like Input fields changing the text colours, background and even the border.

Did It Achieve Expectations

I really like the way that my application turned out, It honestly came out better than I was expecting and It has given me the confidence boost that I was needing.
There is still plenty room for improvement hopefully I can start adding in more advanced features sometime in the future.
