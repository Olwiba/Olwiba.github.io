JavaScript Language

parentheses ()

Parentheses are used in Javascript to provide conditions or pass in parameters.

  • Functions

  • Parentheses are used after the functions name to pass in values for the function to use.
  • function myFunction(value1, value2) {...}

  • Invoking functions

  • When calling a function parentheses are used to pass in the data that the function needs to execute.
  • myFunction(value1, value2);

  • If/Else statements

  • Parentheses are used to create conditions that then true will complete a set of instructions otherwise it will complete a different set of instructions.
  • if (myVar === 0) { do this..} else {do this..}

  • For In loops

  • Parentheses are used to give loops conditions that they will loop though to execute the same code various times.
  • for (task in work.tasks) {...}

  • Specifying values for methods

  • When calling a JavaScript method you need to pass it values that it will need to execute.
  • myString.slice(beginning, end);

Brackets []

In JavaScript brackets are used to create an array, an array is just a container that stores multiple values in a single variable. Unlike many other languages JavaScript lets you store multiple different types of values within the same array. Each value is assigned an index value starting from 0 incrementing upwards for each item and then this index is then used to access the data.

  • Brackets have many uses when working with arrays, Here is how they are used:

  • Syntax for creating an array:
  • var myArray = [data1, data2 ....];
  • Accessing data in an array:
  • myArray[0] = "%data%";

  • As well as creating and accessing arrays brackets can also be used for accessing data inside an object.

  • Bracket notation syntax:
  • myObject["property"]

Braces {}

Braces are often used in JavaScript for defining objects, however we also see that braces are used to encapsulate functions. This is because functions are just special types of objects!

  • The syntax for creating an object in JavaScript would look something like this:

  • var myObject = {
    "property1" : "value1",
    "property2" : "value2"
    .....
    ..
    .
    }

Single quotes '' & double quotes ""

In JavaScript the use of single quotes and double quotes are very much the same, Usually it's the programmers choice to which they prefer although it's best to stick with one or the other to make your code consistent. When working with JSON (JavaScript Object Notation) strings should be double quoted, even though there are many libraries out there that support single quotes there are some that require you to use double. In order to stop yourself running in to simple errors It's best practice to always use double.

If you wanted to have quotes show inside of stings you can use a combination of single and double to achieve this without needing to escape them.
